# Attendly (Vercel Edition)

> Submit permission letters transparently and maintain attendance integrity.

A serverless web application for managing student permission letters related to attendance. Students upload letters directly to cloud storage, and admins manage everything via a secure dashboard.

## ✨ Features

### 🎓 Student Portal
- **Cloud-based uploads**: Seamlessly powered by **Vercel Blob**.
- **Duplicate Prevention**: Integrated real-time check against Supabase to prevent multiple daily submissions.
- **Smart Receipts**: Instant submission confirmation with local date/time formatting.
- **Responsive UI**: Clean, modern design that works perfectly on mobile and desktop.

### 🛡️ Admin Dashboard
- **Secure Access**: JWT-based authentication for a robust and persistent session.
- **Powerful Analytics**: Real-time counters for daily submissions and unique student counts.
- **Advanced Management**: Search, date-based filtering, and instant CSV exports.
- **Media Tools**: Direct file preview and bulk deletion capabilities.
- **Automated Maintenance**: Daily cleanup cron job to keep storage and database costs low.

## ⚙️ Setup & Installation

### 1. Prerequisites
- [Vercel CLI](https://vercel.com/docs/cli) installed (`npm install -g vercel`)
- [Supabase](https://supabase.com) account and project

### 2. Local Environment
Clone the repository and install dependencies:
```bash
npm install
```

Create a `.env` file in the root:
```env
SUPABASE_URL=your_supabase_url
SUPABASE_ANON_KEY=your_supabase_anon_key
BLOB_READ_WRITE_TOKEN=your_vercel_blob_token
JWT_SECRET=your_secure_random_string
ADMIN_USER=admin
ADMIN_PASS_HASH=bcrypt_hashed_password
CRON_SECRET=your_cron_verification_secret
```

### 3. Database Schema
Run the following SQL in your Supabase SQL Editor:
```sql
CREATE TABLE submissions (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT timezone('utc'::text, now()) NOT NULL,
  roll_number TEXT NOT NULL,
  name TEXT,
  reason TEXT,
  file_url TEXT NOT NULL,
  original_name TEXT
);

-- Index for faster daily lookups
CREATE INDEX idx_submissions_roll_date ON submissions (roll_number, created_at);
```

### 4. Running Locally
```bash
npm run start-dev
```
Access the application at `http://localhost:3000`.

## 🧹 Automated Maintenance
The project includes a built-in cleanup cron job configured in `vercel.json` and implemented in `api/cron/cleanup.js`.

- **Schedule**: Runs every day at 00:00 UTC.
- **Action**: Automatically deletes submission records and their corresponding Vercel Blob files if they are older than 24 hours.
- **Security**: Protected via `CRON_SECRET` header verification.
